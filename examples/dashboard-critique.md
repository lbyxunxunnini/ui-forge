# Dashboard Critique Example

## User Input

```text
uif-重设计这个后台首页，先别出代码。先告诉我为什么它看起来像模板，再给一个新的结构方向。
```

## Expected Shape

```text
[ui-forge] 需求分析师：
- 当前目标：不是直接实现，而是先做诊断和重设计方向收口
- 页面类型：后台首页 / dashboard
- 输出模式：诊断模式
- 待确认：首屏最重要的业务动作是什么

[ui-forge] UI设计师：
- 整体诊断：问题不是配色，而是 figure-ground 失效；顶部统计卡、筛选区和主表格同时争抢注意力
- 结构问题：
  1. 首屏没有单一焦点
  2. 卡片和阴影层级过多
  3. 信息密度不统一
- 新方向：
  1. 顶部改成 1 个业务总览 + 1 个主 CTA
  2. 筛选区压缩为 48px 工具条
  3. 表格与趋势区做 7:5 结构切分
- 不建议做的事：
  1. 不要继续叠卡片阴影
  2. 不要把所有统计模块都做成同等权重
```

## What Good Looks Like

- 先诊断，再提方案
- 能解释“像模板”的结构性原因
- 修正建议必须落到布局、层级或组件动作
- 明确写出不建议做的事
