# Skill 可见性规则

## 角色输出标注

每个角色的关键决策和建议必须带角色标签。

### 标注格式

```
[ui-forge] 角色名：
- 决策/建议内容
- 原因/理由
```

### 示例

```
[ui-forge] 需求分析师：
- 平台选择：iOS
- 原因：用户明确指定iOS平台

[ui-forge] UI设计师：
- 风格选择：现代风格
- 原因：渐变背景和卡片布局符合现代风格特征
```

## 输出规则

### 角色输出

- 按顺序逐角色输出
- 每个角色输出带角色标签
- 不合并多个角色到一段输出

### Relentless追问输出

```
[ui-forge] 需求分析师：第N轮追问
- 问题描述？
  A. 选项1（说明）
  B. 选项2（说明）
  C. 选项3（说明）
  推荐：选项X（推荐理由）
```

```
[ui-forge] UI设计师：第N步 - （步骤名称）
- 方案A：（方案描述）
- 方案B：（方案描述）
- 方案C：（方案描述，可选）
- 推荐：方案X（推荐理由）
- 请选择方案，我将继续追问该方案的设计细节
```

## 禁止事项

1. **禁止跳过角色**：不能跳过前面角色直接给结论
2. **禁止合并角色**：不能合并多个角色到一段输出
3. **禁止自动补默认值**：不能自动补齐需求空白
4. **禁止假装看懂设计图**：不能假装理解设计图内容
5. **禁止一次问多个问题**：必须一次只问一个问题
6. **禁止只给一个方案（L3/L4）**：L3/L4 场景下 UI设计师必须提供2-3个方案；L1/L2 默认给1个推荐方案
7. **禁止逻辑未闭环**：需求分析师必须追问到逻辑闭环
8. **禁止设计未闭环**：UI设计师必须追问到设计闭环
