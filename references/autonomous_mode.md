# Autonomous Mode (uif-a)

## 触发条件

用户输入以 `uif-a`、`uif a` 或 `uif_a` 开头时进入。

## 核心理念

用户不想被追问，想让系统自己做合理决策。autonomous 模式用推荐方案和 `auto_assumption` 补齐缺失信息，只在高风险决策时才确认。

## 与标准模式的区别

| 维度 | 标准模式 (uif-) | 自动模式 (uif-a) |
|------|----------------|-----------------|
| 提问 | 按预算逐轮追问 | 默认不问，用推荐方案补齐 |
| 确认 | 每个阶段都确认 | 只在高风险时确认 |
| 方案选择 | 用户选择方案 | 系统选择推荐方案 |
| 速度 | 正常 | 快（减少交互轮次） |
| 控制权 | 用户主导 | 系统主导，用户兜底 |

## auto_assumption 机制

当信息不足时，系统自动使用推荐值并标注为 assumption：

```
[ui-forge] controller：自动补齐以下信息
- 平台：iOS（从上下文推断）
- 风格：现代（默认推荐）
- 主色：#4F46E5（行业常用）
- 标注：以上为自动假设，如需调整请告知
```

### 可自动假设的项目

| 项目 | 默认值 | 推断来源 |
|------|--------|----------|
| 平台 | iOS | 项目上下文 |
| 风格 | 现代 | 最常见选择 |
| 主色 | #4F46E5 | 行业常用 |
| 字体 | -apple-system, sans-serif | 平台默认 |
| 圆角 | 8px | 现代风格常用 |
| 间距 | 16px 基数 | 标准间距系统 |
| 深色模式 | 否 | 默认浅色 |
| 响应式 | 是 | 默认包含 |

### 不能自动假设（必须确认）

| 项目 | 原因 |
|------|------|
| 页面核心功能 | 功能错误代价太高 |
| 整体风格重构 | 影响所有后续设计 |
| 多页面路由关系 | 架构决策不可逆 |
| 设计系统核心 token | 影响所有页面一致性 |

## 高风险确认触发

当遇到以下情况时，即使在 autonomous 模式也必须确认：

```
[ui-forge] controller：检测到高风险决策，需要确认
- 决策：（具体决策内容）
- 风险：（为什么高风险）
- 选项A：（方案A，推荐）
- 选项B：（方案B）
- 请确认
```

触发条件：

1. 用户需求有多种合理解读
2. 设计决策会导致不可逆的架构变化
3. 涉及品牌/风格的根本性选择
4. 涉及多页面一致性约束

## 流程

```
用户输入需求
  → 读取 design card（如存在）
  → 自动补齐缺失信息（auto_assumption）
  → 完整设计流程（需求分析 → UI设计）
  → 只在高风险时暂停确认
  → 输出设计文件
  → 自动退出
```

## 输出格式

### 自动假设汇总

```
[ui-forge] controller：自动模式启动
- 任务：（任务描述）
- 自动假设：
  · 平台：iOS
  · 风格：现代
  · 主色：#4F46E5
- 如需调整以上假设，请随时告知
```

### 设计输出

与标准模式相同，但省略方案选择环节，直接输出推荐方案。

## 示例

### 输入
```
uif-a设计一个后台仪表盘，数据密集型，深色主题
```

### 输出流程

```
[ui-forge] controller：自动模式启动
- 任务：后台仪表盘设计
- 已明确：数据密集型、深色主题
- 自动假设：
  · 平台：Web（后台通常为 Web）
  · 布局：侧边栏 + 主内容区（后台常用）
  · 断点：1280px/768px
  · 密度：compact（数据密集型）

[ui-forge] 需求分析师：需求收口完成（自动模式）
- 页面类型：仪表盘
- 数据模块：概览卡片、图表区、数据表格、筛选工具栏
- 状态：loading、empty、error、正常
- 自动补齐：主色 #4F46E5（行业常用），间距 16px 基数，圆角 8px

[ui-forge] UI设计师：设计方案
- 布局：左侧 240px 侧边栏 + 右侧主内容区
- 主内容：顶部概览卡片（4列）→ 中部图表 → 底部表格
- tokens.json 已锁定
- 输出文件...
```
