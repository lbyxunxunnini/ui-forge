# 上游返回机制

任何角色发现自己需要的上游输入缺失或不明确时，可以向对应上游角色返回，要求补充或澄清。这是比讨论回合更轻量的机制——不要求"分歧"，只要求"缺失"。

## 返回格式

```
[ui-forge] UI设计师：返回 需求分析师
- 缺失内容：按钮点击后的加载状态和成功/失败反馈
- 请补充：加载中的样式、成功后的跳转逻辑、失败时的提示方式
```

## 返回规则

1. 返回只允许向上游返回，不允许跨角色返回
2. 返回后，被返回角色必须响应，不允许忽略
3. 响应格式与返回格式相同，只是把"缺失内容"改为"补充内容"
4. 一轮任务最多返回 2 次。超过 2 次仍未解决，由需求分析师拍板

**与讨论回合的关系：** 返回机制（缺失补充）和讨论回合（分歧解决）是独立计数的。返回用完 2 次后仍可进入讨论回合（最多 4 轮），反之亦然。两种机制的总交互轮次上限为 6 轮（2 次返回 + 4 轮讨论），超过后由需求分析师拍板。

## 返回示例

### UI设计师返回需求分析师

```
[ui-forge] UI设计师：返回需求分析师
- 缺失内容：登录成功后的跳转页面
- 请补充：登录成功后跳转到首页还是之前的页面？
```

### 需求分析师响应

```
[ui-forge] 需求分析师：补充内容
- 补充内容：登录成功后跳转到首页
- 原因：首次登录用户需要看到首页内容
```
