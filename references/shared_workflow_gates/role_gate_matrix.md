# Role Gate Matrix

## 需求分析师

- 进入：从需求、重设计目标、页面规划或设计诊断开始
- 停机：关键逻辑、内容来源、主 CTA 或边界态未补全
- 放行：用户确认状态允许进入 UI 设计阶段
- 建议预算：`L2 2-4` 问，`L3 4-7` 问，`L4 6-10` 问分阶段执行

## UI 设计师

- 进入：需求已确认，且具备可靠视觉输入或足够清晰的文字结构
- 停机：视觉输入不可读，或多种拆法都合理且会影响结构方向
- 放行：布局、组件边界、交互状态足够支撑交付或前端实现
- 建议预算：`L2 1-2` 次确认，`L3 2-4` 次确认，`L4 3-5` 次确认

## 交付阶段

- 进入：需求和 UI 结构已足够稳定，适合生成 HTML/CSS/SVG、tokens 和文档
- 停机：仍存在会影响页面形态、组件状态或设计系统的未决项
- 放行：产物结构、设计说明、状态覆盖和完整性检查通过
- 预算原则：只确认阻塞交付的缺口，不为仪式感追加问题
