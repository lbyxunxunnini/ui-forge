# 记忆功能

**重要规则：每次完成一轮大设计都需要把进度保存成文档，以便下次进入时读取。一轮任务完成即退出，不常驻。**

## 三个体系分离（核心）

**设计规则卡、内部记忆和设计输出是三个独立体系：**

### 体系0：设计规则卡 `.ui-forge/projects/`（项目级）

**用途：** 长期 UI 规则，跨 session 持久
**用户可见：** 是（但通常不主动展示）
**详见：** [design_card_protocol.md](design_card_protocol.md)

### 体系1：内部记忆 `.design-doc/`（用户不可见）

**用途：** 进度追踪、需求记录、设计摘要
**用户可见：** 否

### 体系1：内部记忆 `.design-doc/`（用户不可见）

**用途：** 进度追踪、需求记录、设计摘要
**用户可见：** 否

**内容：**
- 项目总目录（README.md）
- 模块设计文档（需求总结、设计进度、变更记录）
- 设计规范摘要（颜色、字体、间距等）

### 体系2：设计输出 `/design-output/`（用户可见）

**用途：** 实际设计文件
**用户可见：** 是

**内容：**
- HTML/CSS 文件
- 设计Token文件
- SVG图标（所有用到的图标都要导出）
- 截图/效果图
- 响应式版本
- 设计规范文档

**图标导出规则：**
- 所有图标必须单独导出为SVG文件
- 存放在 `design-output/icons/` 目录
- 命名规范：英文小写+连字符（如 `phone.svg`、`wechat.svg`）
- 无论HTML中使用内嵌SVG还是图片引用，都要单独导出文件

## 向用户展示规则

**设计完成后，只展示 `/design-output/` 的内容，不展示 `.design-doc/` 的内容：**

```
---
设计输出

文件位置：design-output/index.html
打开方式：在浏览器中直接打开即可预览

设计文件
- design-output/index.html — 主页面
- design-output/style.css — 样式文件
- design-output/icons/ — SVG图标
- design-output/screenshots/ — 效果图
- design-output/DESIGN-GUIDE.md — 设计规范

如需调整任何细节，直接告诉我即可。
---
```

## 记忆文件结构

**按模块分组，支持同类型多页面：**

```
.design-doc/
├── README.md                    # 项目总目录
├── auth/                        # 认证模块
│   ├── _index.md               # 模块总览
│   ├── login-regular.md        # 常规登录
│   ├── login-oneclick.md       # 一键登录
│   └── login-sms.md            # 验证码登录
├── chat/                        # 聊天模块
│   ├── _index.md               # 模块总览
│   ├── chat-main.md            # 聊天主页
│   └── chat-detail.md          # 聊天详情
├── settings/                    # 设置模块
│   ├── _index.md               # 模块总览
│   ├── settings-main.md        # 主设置页
│   └── settings-account.md     # 账户设置
└── home/                        # 首页模块
    ├── _index.md               # 模块总览
    └── home-main.md            # 首页
```

### 目录命名规则

1. **模块目录**：使用英文小写，简短明确（如 `auth`、`chat`、`settings`）
2. **页面文件**：`页面类型-变体名称.md`（如 `login-regular.md`）
3. **模块索引**：`_index.md`（下划线开头，便于排序）

### 模块划分原则

按功能域划分，不按页面类型划分：

| 模块 | 包含页面 |
|------|----------|
| auth | 登录、注册、找回密码、验证码 |
| chat | 聊天列表、聊天详情、群聊 |
| settings | 主设置、账户设置、隐私设置 |
| home | 首页、Dashboard |
| profile | 个人中心、编辑资料 |
| search | 搜索、筛选 |
| detail | 商品详情、文章详情 |
| list | 列表页、分类页 |

### 变体命名规则

同一页面类型的不同设计变体：

| 页面类型 | 变体示例 |
|----------|----------|
| 登录 | regular（常规）、oneclick（一键）、sms（验证码）|
| 首页 | main（主首页）、feed（信息流）、dashboard（仪表盘）|
| 设置 | main（主设置）、account（账户）、privacy（隐私）|
| 列表 | grid（网格）、list（列表）、card（卡片）|
| 详情 | full（全屏）、modal（弹窗）、panel（侧面板）|

## 项目隔离规则（核心）

**禁止跨项目读取设计记忆或规则卡：**

1. `.design-doc/` 属于当前项目，不得读取其它项目的 `.design-doc/`
2. `.ui-forge/projects/<project>.design_card.yaml` 属于特定项目，不得用于其它项目
3. `design-output/` 属于当前项目，不得将其它项目的输出当作当前项目的参考
4. 如用户切换到新项目，必须从空白状态开始，除非用户明确说"沿用之前的风格"

**违反隔离时：**
```
[ui-forge] controller：项目隔离冲突
- 当前项目：（项目名）
- 检测到读取：（其它项目的路径）
- 操作：已忽略跨项目数据，从当前项目状态开始
```

## 保存时机

满足以下任一条件时，保存记忆文档：

1. **完成需求分析**：需求分析师确认需求后
2. **完成UI设计**：UI设计师输出设计方案后
3. **用户确认设计**：用户确认设计方案后
4. **退出时**：用户要求退出时

## 设计系统锁定（项目级）

**当项目包含3+页面时，设计系统锁定机制自动激活：**

### 锁定时机

第一个页面设计完成、用户确认后，tokens.json锁定为项目设计系统。

### 锁定内容

```
.design-doc/
├── README.md
├── design-system-lock.md    # 设计系统锁定记录
├── auth/
│   ├── _index.md
│   └── login-regular.md
└── ...
```

### 锁定记录格式

```markdown
# 设计系统锁定

## 锁定信息
- 锁定时间：2026-05-14
- 锁定页面：login-regular（首个完成页面）
- 锁定文件：design-output/tokens.json

## 锁定值
- 主色：#667eea
- 辅助色：#764ba2
- 背景色：#0a0a1a
- 字体：-apple-system, BlinkMacSystemFont, sans-serif
- 间距基数：4px

## 后续页面约束
- 所有后续页面必须复用以上值
- 如需修改，必须记录变更并通知所有已设计页面
```

### 修改流程

```
用户要求修改设计系统
  → UI设计师评估影响范围（哪些页面需要同步修改）
  → 列出受影响页面清单
  → 用户确认修改
  → 更新 tokens.json
  → 更新 design-system-lock.md
  → 同步修改所有受影响页面
```

## 保存规则

**记忆文档保存到 `.design-doc/`，设计输出保存到 `/design-output/`：**

### 记忆文档保存规则

1. **更新总目录**：每次保存时更新 `.design-doc/README.md`
2. **创建/更新模块索引**：如模块不存在，创建 `.design-doc/模块名/_index.md`
3. **创建/更新页面文档**：创建或更新 `.design-doc/模块名/页面类型-变体.md`
4. **记录变更**：在变更记录中记录本次变更
5. **更新进度**：更新设计进度状态
6. **不保存代码**：记忆文档只保存设计摘要，不保存HTML/CSS代码

## 记忆检查与覆写规则（核心）

### 检查时机

**用户发起新设计时，必须先检查记忆：**

1. 读取 `.design-doc/README.md` 总目录
2. 检查是否存在同名模块
3. 如存在，提醒用户并询问是否重新设计

### 检查流程

```
[ui-forge] 需求分析师：检测到已有设计
- 模块名称：登录模块
- 模块路径：.design-doc/auth/
- 已有设计：login-regular.md（2026-05-14 创建，现代风格，iOS平台）
- 设计状态：已完成
- 选项A：重新设计（推荐，将覆写已有设计）
- 选项B：继续使用已有设计
- 选项C：查看已有设计详情
请确认：
```

### 模块结构校验

**保存记忆时必须校验目录结构：**

1. **禁止文件散落在根目录** — 所有页面文档必须在模块子目录中
2. **模块目录必须有 `_index.md`** — 索引文件记录模块内所有页面
3. **总目录 `README.md` 必须更新** — 新增页面必须在页面清单中登记

**校验不通过时：**
```
[ui-forge] 需求分析师：记忆结构异常
- 问题：（如：页面文档散落在根目录）
- 修正：正在整理为模块化结构
```

### 覆写规则

**只有用户明确确认后才能覆写记忆：**

1. **用户选择"继续使用"**：读取已有设计，恢复上下文，继续设计流程
2. **用户选择"重新设计"**：确认覆写，清除旧记忆，重新开始设计
3. **用户选择"查看详情"**：展示已有设计详情，再次询问

### 覆写格式

```
[ui-forge] 需求分析师：确认覆写
- 覆写模块：auth（认证模块）
- 覆写页面：login-regular（常规登录）
- 原设计：2026-05-14 创建，现代风格
- 新设计：（新设计信息）
- 确认状态：已确认覆写
- 操作：清除旧记忆，重新开始设计
```

### 覆写操作

1. **备份旧文件**（可选）：将旧文件重命名为 `.backup-日期-页面名.md`
2. **清除旧记忆**：删除或重命名旧页面文档
3. **更新模块索引**：从 `_index.md` 中移除旧记录
4. **更新总目录**：从 `README.md` 中更新页面清单
5. **重新设计**：从头开始设计流程

### 覆写示例

**覆写登录页设计：**
