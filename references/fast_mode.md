# Fast Mode (uif-fast)

## 触发条件

用户输入以 `uif-fast` 或 `uif fast` 开头时进入。

也可由标准入口 `uif-` 自动路由：当任务满足以下全部条件时命中 fast 模式：

1. 有明确修改目标（颜色、字号、间距、圆角、图标等）
2. 不涉及流程变化或信息架构调整
3. 修改范围局部，不影响整体布局

## 核心规则

| 规则 | 说明 |
|------|------|
| 跳过需求分析师 | 直接进入 UI设计师 |
| 0-1 问 | 不追问，除非修改有歧义 |
| 不走审美诊断 | 不调用 polanyi-design |
| 不给多方案 | 直接执行，不提供选项比较 |
| 任务完成即退出 | 输出修改文件后自动退出 |

## 可处理的调整类型

| 类别 | 示例 |
|------|------|
| 颜色 | 把主按钮改成 #4F46E5 |
| 字号 | 标题改成 24px |
| 间距 | 卡片内边距改成 20px |
| 圆角 | 按钮圆角改成 8px |
| 字体 | 换成 SF Pro |
| 图标 | 把锁图标换成盾牌图标 |
| 阴影 | 去掉卡片阴影 |
| 对齐 | 按钮右对齐 |

## 不能处理（应回退到标准模式）

- 新增页面或功能
- 修改交互流程
- 重设计布局
- 涉及多页面一致性
- 用户说"重新设计"

遇到以上情况时：

```
[ui-forge] UI设计师：此修改涉及布局/流程变化，建议使用标准模式。
- 当前修改：（描述）
- 影响范围：（超出 fast 模式范围的部分）
- 建议：使用 uif- 进入标准流程
- 是否继续 fast 模式？（可能需要简化处理）
```

## 输出格式

```
[ui-forge] UI设计师：快速调整完成
- 修改项：（具体改了什么）
- 影响文件：（哪些文件被修改）
- 变更前后：（关键值对比）
```

## 最小输出集

快速模式不需要执行完整的输出完整性检查清单，最小输出集为：

- `style.css` — 修改后的样式文件
- `tokens.json` — 更新后的设计 token（如有变更）
- 无需输出 `REQUIREMENTS.md`、`icons/`、`DESIGN-GUIDE.md`

## 示例

### 输入
```
uif-fast把主按钮改成#4F46E5
```

### 输出
```
[ui-forge] UI设计师：快速调整完成
- 修改项：主按钮背景色
- 变更：#667eea → #4F46E5
- 影响文件：style.css, tokens.json
- 影响检查：无布局/响应式/交互影响
```
