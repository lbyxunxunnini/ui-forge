# 测试用例

## 测试用例1：登录页设计

### 输入

```
uif-设计一个iOS登录页，现代风格，渐变背景，包含微信登录
```

### 预期输出

1. 需求分析师输出需求分析
2. 用户确认需求
3. UI设计师输出设计方案
4. 用户确认设计
5. 输出设计文件

### 验证点

- [ ] 需求分析师正确识别平台（iOS）
- [ ] 需求分析师正确识别页面类型（登录页）
- [ ] 需求分析师正确识别风格（现代风格）
- [ ] 需求分析师正确识别功能（微信登录）
- [ ] UI设计师输出完整的设计方案
- [ ] 设计文件包含HTML、CSS、Token

## 测试用例9：设计诊断模式

### 输入

```text
uif-重设计这个后台首页，先别出代码。先告诉我为什么它看起来像模板，再给一个新的结构方向。
```

### 预期输出

1. 需求分析师识别当前任务是诊断模式，不直接进入交付
2. UI设计师给出整体诊断
3. UI设计师指出结构问题和新方向
4. 输出中包含明确禁止项
5. 提问预算命中 `L3`

### 验证点

- [ ] 正确识别为诊断模式
- [ ] 能解释“像模板”的结构性原因
- [ ] 修正建议落到布局、层级或组件动作
- [ ] 包含明确的不建议做的事
- [ ] 没有退化成固定八轮问答

---

## 测试用例10：设计系统收口

### 输入

```text
uif-我们已经有 4 个页面了，帮我收口一套可复用的设计系统，不要重新画页面。
```

### 预期输出

1. 识别任务属于设计系统收口
2. 优先收口 type scale、spacing scale 和 semantic tokens
3. 提取共享组件和状态规范
4. 输出结果可供前端实现使用
5. 提问预算命中 `L4`

### 验证点

- [ ] 没有直接跳到单页视觉稿
- [ ] 先收口 tokens 再收口组件
- [ ] 覆盖组件状态而不是只覆盖静态样式
- [ ] 产物结构可交付
- [ ] 提问按里程碑分段，而不是一次问满

---

## 测试用例11：常规单页的中等预算

### 输入

```text
uif-设计一个 iOS 设置页，简洁风格，包含通知、隐私和退出登录。
```

### 预期输出

1. 识别任务属于常规单页
2. 提问预算命中 `L2`
3. 只追问 2-4 个会影响结果的关键问题
4. 不进入系统级长流程

### 验证点

- [ ] 正确命中 `L2`
- [ ] 没有追问与结果无关的分支
- [ ] 平台、核心结构和主要交互已收口
- [ ] 没有被拉到 `L3/L4`
